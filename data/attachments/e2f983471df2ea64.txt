# Failure Triage Summary

## Defect Title

POST /carts/add assigns incorrect quantities to products (quantities appear swapped between products)

## Category

API behavior change / server logic bug

## Expected Result

When POSTing a cart with products [{id:144, quantity:4}, {id:98, quantity:1}], the response should show product id 144 with quantity 4 and product id 98 with quantity 1.

## Actual Result

Response shows product id 144 with quantity 1 and product id 98 with quantity 4 (quantities appear swapped). Response status 201 and totals reflect these swapped quantities.

## Suspected Root Cause

Server-side mapping logic is assigning quantities to the wrong product entries in the response (e.g., an index-alignment bug or products being reordered server-side before quantities are associated), resulting in quantities being attributed to the incorrect product ids.

## Reproduction Steps

1. Send POST request to https://dummyjson.com/carts/add with Content-Type: application/json and body: { "userId": 1, "products": [ { "id": 144, "quantity": 4 }, { "id": 98, "quantity": 1 } ] }
2. Observe the 201 response body and inspect the 'products' array in the response.
3. Verify that product with id 144 has quantity 1 while product with id 98 has quantity 4 (i.e., quantities are swapped compared to the request).
4. Repeat the POST (or change the order/quantities) to confirm the issue is reproducible and consistent.

## Recommended Next Debugging Action

Triage with backend engineers: provide the exact request and response payloads and ask them to inspect the server-side handling of the incoming 'products' array. Check code paths that map requested quantities to product records for index-based mapping, sorting, or mutation of the products list. Reproduce the request in a client (Postman/cURL) and capture server logs for the request to confirm how the server parsed and associated quantities. Add or run unit/integration tests covering multiple products to validate correct id->quantity association.
